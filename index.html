Why You Should Give Me a 4/4 For This Assignment?

This assignment implement a icosphere mesh generation algorithm with mesh subdivision. Some articles online (like this one) talked about some implementation detail. BUT, most articles focused on generating mesh instead of texture coordinate generation.

To generate a icosphere mesh with possible texture coordinate mapped to a real world map. A lot of other things need to be done. The most important one is guaranteeing that there's no mesh go across the poles. Since it will be difficult to map the texture to the mesh at poles.

To generate texture coordinates for the mesh, I calculate the longitude and latitude of each vertex. But this makes the texture mapping near the International Date Line not correct. Some extra work is done here to make everything looks good.

Finally, this assignment implement a specular map to make the land looks more rough and water looks more shiny. The framework actually supports more texture maps if you want.

4. Change the mode on the top right corner from "move" to "view", now you can use left click to move the camera. 5. Change the mode to "rotate", now you can drag the handles to rotate the selected entity.

Mesh Subdivision Usage:
1. Create a new entity.
2. Add TransformData3D, IcosphereMeshGeneratorData, and MeshRenderData3D to the entity.
3. Change the flatNormal field to true for flattened normals.
4. Change the subdivision field to the number of subdivisions.
5. Change the radius field to the radius of the sphere.
6. You may not change the subdivision number too large. Set to 3 or 4 is recommended.
